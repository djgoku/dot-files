* Overview

Personal development environment configuration using [[https://mise.jdx.dev/][mise]] for tool management and [[https://nixos.org/][Nix]] for package management. Features an Emacs configuration with custom overlay build and integrated development tools.

* Quick Start

** Prerequisites

- macOS (Darwin)
- Command Line Tools for Xcode: =xcode-select --install=
- Internet connection

** Automated Setup

Run the idempotent setup script:

#+begin_src sh
curl -fsSL https://raw.githubusercontent.com/djgoku/dot-files/main/setup.sh | zsh
#+end_src

Or clone and run locally:

#+begin_src sh
mkdir -p ~/dev/github/djgoku && cd "$_"
git clone https://github.com/djgoku/dot-files.git
cd dot-files
./setup.sh
#+end_src

** Customization

Clone a specific branch via environment variable:

#+begin_src sh
# Clone a specific branch instead of the default
REPO_BRANCH="feature-branch" ./setup.sh
#+end_src

Default: =REPO_BRANCH=""= (uses repository's default branch)

** macOS Configuration

The setup applies these macOS defaults:
- Enable Dark mode
- Disable natural scroll direction
- Set Finder to list view
- Show all files in Finder
- Import Terminal preferences (if =com.apple.Terminal.plist= exists)

* Post-Installation

After setup completes:

1. **Activate mise in your shell:**

   #+begin_src sh
   eval "$(~/.local/bin/mise activate zsh)"
   #+end_src

   Or add to =~/.zshrc= (already done by =setup-global= task):

   #+begin_src sh
   eval "$(~/.local/bin/mise activate zsh)"
   #+end_src

2. **Verify installation:**

   #+begin_src sh
   mise which git
   mise which Emacs
   mise which node
   #+end_src

3. **Launch Emacs:**

   #+begin_src sh
   mise run emacs          # GUI mode
   mise run emacs --nw     # Terminal mode
   #+end_src

* Usage

** mise Tasks

Common tasks defined in [[file:mise.toml][mise.toml]]:

*** Launch Emacs

#+begin_src sh
mise run emacs           # GUI mode
mise run emacs --nw      # Terminal mode (no window)
#+end_src

*** Generate Emacs Environment Paths

Creates =.env= file with tool paths for Emacs:

#+begin_src sh
mise run emacs-paths
#+end_src

*** Select Emacs Environment

Interactive environment selector (detects SSH sessions):

#+begin_src sh
mise run emacs-env-select
mise run emacs-env-select --nw  # Force terminal mode
#+end_src

*** Install Enchant (Manual)

Enchant requires both development headers and runtime library:

#+begin_src sh
mise run install-enchant
#+end_src

*** Setup Global Configuration

Install mise config globally to =~/.config/mise/config.toml=:

#+begin_src sh
mise run setup-global
#+end_src

** Environment Variables

The =.env= file (generated by =emacs-paths= task) contains:

#+begin_example
MISE_VTERM_PATH=/nix/store/.../vterm
MISE_ENCHANT_INCLUDE_PATH=/nix/store/.../enchant^dev
MISE_ENCHANT_LIB_PATH=/nix/store/.../enchant
#+end_example

* Troubleshooting

** Setup Script Issues

The setup script is idempotent and can be safely re-run. Each step checks for existing installations.

** mise Issues

Check mise status:

#+begin_src sh
mise doctor
mise config ls
#+end_src

View mise cache:

#+begin_src sh
mise cache clear
#+end_src

** Emacs Launch Failures

1. Verify tool paths:

   #+begin_src sh
   cat .env
   #+end_src

2. Regenerate paths:

   #+begin_src sh
   mise run emacs-paths
   #+end_src

3. Check Emacs binary:

   #+begin_src sh
   mise which Emacs
   $(mise which Emacs) --version
   #+end_src

** GPG Agent Issues

If GPG/SSH agent fails to start, ensure =gpgconf= is available:

#+begin_src sh
gpgconf --list-dirs
gpgconf --launch gpg-agent
#+end_src
